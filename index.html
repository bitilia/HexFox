About

Promptathon is competition management software built by students for running AI literacy events at schools. It covers the entire event: participant registration, a built-in learning Academy, secure file submissions, structured judging with weighted rubrics, and GDPR-compliant data handling.

We built it because running a good AI competition is hard work and no decent tool existed for schools. We want every school to be able to run one regardless of budget.

Why does student data privacy matter here?

Promptathon is designed to be entirely self-hosted so data never leaves the institution responsible for it. During alpha we host it carefully on UK servers in per-school isolated databases. After alpha, any school can run Promptathon independently on their own server with zero access from us.

Features

  • Built-in Academy — self-paced modules covering AI fundamentals, prompt engineering, AI safety, and responsible use. Participants must complete these before they can submit.
  • Configurable tasks — multiple competition briefs at different difficulty levels. Participants choose their task at submission time, not at registration.
  • Weighted judging rubrics — define scoring criteria with custom weights. Judges score independently through a dedicated interface. Results aggregate automatically. No spreadsheets.
  • Secure file submissions — PDF and DOCX accepted. DOCX is converted to PDF by an isolated process with no network access. Every file is structurally verified and sanitised before storage.
  • OTP email login — no passwords. Participants log in with a one-time code sent to their email. Registration can be restricted to your school's email domain.
  • Live security dashboard — real-time connection logs, login failure rates, suspicious file counts, active IP tracking, country restrictions, and IP blocking.
  • Emergency lockdown — one click to halt all registrations and submissions site-wide.
  • GDPR data request portal — participants can submit access, export, and deletion requests through a self-service portal at any time. All requests are logged.
  • Immutable audit log — every significant action is recorded with timestamp, actor, and IP address.
  • Admin content editor — edit every page title, body text, and FAQ from the admin panel without touching code. Theme editor for colours and branding.
  • One-command deployment — post-alpha, a single Python script installs Caddy (automatic HTTPS), Gunicorn, UFW firewall, email delivery, and all services on any Ubuntu 22.04 server.

Hosting model

Now (alpha) — we host it

We provision your Promptathon instance on our infrastructure and hand you the keys. You get a subdomain like yourschool.promptathon.uk, or connect your own domain by pointing a DNS record at us. We configure everything; you customise it through the admin panel and run your competition. Free, with no time limit.

After alpha — you own it

Once the platform is stable, any school or organisation can deploy Promptathon independently. One Python setup script on any Ubuntu 22.04 server installs everything: Caddy with automatic HTTPS, the application, firewall rules, email delivery, and all services. Your domain, your database, your server. Zero dependency on us after deployment. Alpha schools get first access and full support migrating to their own server if they want it.

Frequently asked questions

Is it really free?

Yes. No hidden costs, no premium tier, no expiry on alpha access. We are students building this to support AI education. Making money is not the current goal.

Do I need to run a server?

Not during alpha. We host every instance. You contact us, we set it up, you get a link. No servers, no SSH, no configuration. Post-alpha, self-hosting is an option if you want full data sovereignty, using a single setup script on a cheap VPS.

Can I use my own domain?

Yes. By default we give you a subdomain like yourschool.promptathon.uk. If you prefer your own domain, mention it and we set it up. You point a DNS CNAME or A record at our server, which takes two minutes.

Where is student data stored?

During alpha: on our servers in a per-school isolated database on UK infrastructure. We treat all data as strictly confidential and do not access it except when explicitly asked for support. Post-alpha, self-hosted: entirely on your own server with zero access from us. That is the long-term design goal.

Is it GDPR compliant?

Yes. Built-in data request handling lets participants submit access, export, and deletion requests through a self-service portal. All requests are tracked and logged. No telemetry, no analytics, no external data sharing.

How many students can it handle?

Comfortably hundreds on a modest server. The platform uses SQLite with WAL mode for concurrent access and threaded Gunicorn workers. For 300 or more simultaneous submissions we recommend at least a 2-core, 4 GB instance.

What happens after the alpha period?

We have not fixed a commercial model. Alpha schools will have plenty of notice before anything changes and will always be credited for helping build this. The core software will remain open source.

Who built Promptathon?

Students who believe AI literacy education should be accessible to every school, not only well-funded ones. We built the thing we wished existed and are sharing it freely while we work out what it wants to become.
