1. Who we are and scope

Promptathon is a software project developed and maintained by its student creators (referred to as "we," "us," or "the developers"). This document applies to this marketing website (promptathon.uk), alpha-hosted Promptathon instances we operate on behalf of schools, and direct communications between you and us.

It does not apply to the internal operations of self-hosted Promptathon instances. Schools deploying Promptathon independently are solely responsible for their own privacy obligations and must provide their own privacy notices to participants.

2. Data collected on this website

This is a static HTML page with no database, no user accounts, and no server-side application logic. The only personal data we may process through this website is:

Server access logs

Our web server generates standard access logs recording your IP address, the resource requested, HTTP method, response status code, bytes transferred, and timestamp. These are used exclusively for security monitoring and diagnosing technical issues. Retained for a maximum of 30 days. Never shared with third parties or used for profiling.

Email correspondence

If you contact us at the email address published on this page, we receive and store that communication, including any personal information you include, solely to respond to your enquiry. We do not add you to any mailing list without explicit consent. Retained for a maximum of two years from the last exchange, then deleted. Earlier deletion available on request.

We do not use analytics, tracking pixels, advertising scripts, or any fingerprinting on this website. No personal data beyond the above is collected.

Important: This website is hosted on GitHub Pages and served through Cloudflare. These services are operated by third parties and have their own data collection practices. See Section 9 (Hosting and Infrastructure) for details on what data GitHub Pages and Cloudflare collect.

3. Data on alpha-hosted instances

During the alpha period, we provision and operate Promptathon instances on a Virtual Private Server (VPS) hosted in the EU/UK on behalf of schools. In this arrangement:

  • The school is the data controller. The school determines the purposes and means of processing participant data and is responsible for providing appropriate privacy notices to participants and ensuring a lawful basis for processing.
  • We are the data processor. We process personal data only on the school's instructions and only to the extent necessary to operate, maintain, and support the instance.

Categories of data processed

  • Participant identity: email address (required for login), name or display name if collected by the school.
  • Participation records: Academy module progress, task selection, submission text, uploaded files, timestamps.
  • Judge records: scores and notes attributed to submissions. Judge notes are encrypted at rest using AES-256.
  • Authentication events: login timestamps, OTP issuance and use, IP addresses at login.
  • Connection data: IP addresses, country codes, and HTTP metadata for security monitoring.
  • GDPR request records: data access, export, and deletion requests submitted by participants.
  • Audit log: timestamps, user IDs, and action descriptions for all significant platform events.

How we protect this data

  • Each school's data is stored in a logically isolated database. No cross-school access is possible at the application layer.
  • All data is stored on a VPS hosted in the EU/UK and is not transferred outside the UK or EEA.
  • All connections use HTTPS with TLS 1.2 or higher.
  • We do not access any school's data except when the school explicitly asks us to provide technical support, and only to the minimum extent necessary.
  • We do not sell, license, share, or otherwise disclose school or participant data to any third party for any purpose whatsoever.

4. Data on self-hosted instances

Once a school deploys their own Promptathon instance, we have zero technical access to that instance, its database, or any data it contains. We are not a data controller or processor in relation to that data. The school is the sole data controller and is fully responsible for its own privacy obligations.

The Promptathon software includes built-in tools to help operators meet GDPR obligations: a participant-facing data request portal, subject access request tracking, automated deletion workflows, an immutable audit log, and encrypted storage of sensitive fields.

5. Cookies

This marketing website uses no cookies of any kind.

Alpha-hosted Promptathon instances use a single strictly-necessary session cookie to maintain login state after OTP authentication. This cookie is set only after successful authentication, contains no personal information (only an encrypted session identifier), expires on browser close or after a fixed inactivity period, and is marked HttpOnly, Secure, and SameSite=Lax. It cannot be declined without disabling login. No third-party cookies are set on any Promptathon instance.

6. Lawful basis for processing

Where we process personal data as controller (this website and direct communications):

  • Legitimate interests (Article 6(1)(f) UK GDPR): server access logs for the security and operation of this website.
  • Contract performance (Article 6(1)(b)): email correspondence relating to setting up or supporting an alpha-hosted instance.

Where we process personal data as processor on behalf of a school, processing is carried out under a Data Processing Agreement pursuant to Article 28 UK GDPR. The school as controller determines the lawful basis for each category of processing.

7. Data retention

  • Website server logs: maximum 30 days, then automatically deleted.
  • Email correspondence: maximum two years from the last exchange. Earlier deletion available on request.
  • Alpha instance data: retained for the duration of the school's participation plus a 60-day grace period for data export. Deleted on request at any time, or automatically at the end of the grace period.
  • Alpha instance audit logs: retained for 12 months from the event date.

Schools may request immediate deletion of all data associated with their instance at any time. Deletion is permanent and irreversible; we recommend exporting any data you wish to retain beforehand.

8. Security

We implement the following measures to protect personal data:

  • All connections use HTTPS with TLS 1.2 or higher. HSTS headers are enforced.
  • Sensitive database fields (including judge notes) are encrypted using AES-256 before storage. Encryption keys are stored separately from the data.
  • File processing runs as a dedicated system user with no network access, no shell access, and write access limited to a single directory.
  • All uploaded files are structurally verified and sanitised before storage; active content (JavaScript, macros, embedded scripts) is stripped.
  • UFW firewall rules restrict inbound connections to SSH, HTTP, and HTTPS only.
  • The security dashboard monitors for unusual login patterns, suspicious file uploads, and anomalous connection activity in real time.
  • All access events are logged in an immutable audit trail.
Alpha caveat. Promptathon has not undergone formal independent security audit. While we implement substantive measures, schools processing sensitive student data should conduct a Data Protection Impact Assessment (DPIA) before large-scale deployment and contact us to establish a Data Processing Agreement.

Cloudflare proxy

  • Your IP address and geolocation data (inferred from IP)

Cloudflare may retain this data for up to 30 days for security and analytics purposes. This data is subject to Cloudflare's privacy policy. We do not have control over Cloudflare's data collection practices beyond what is documented in their policy.

12. Data Processing Agreements

Article 28 UK GDPR requires a binding Data Processing Agreement (DPA) between controller and processor. Schools processing personal data on an alpha-hosted instance with us should have a DPA in place before doing so at scale. Contact us at contact@promptathon.uk with the subject line "DPA Request." We will provide a standard DPA covering subject matter, duration, nature and purpose of processing, types of personal data and categories of data subjects, obligations and rights of both parties, security measures, sub-processor disclosure, and procedures for data subject requests, security incidents, and deletion at contract end.
