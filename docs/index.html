About

Mission / Architecture

Every school should be able to run a great AI competition — regardless of budget.

What it is

Competition management software built by students for AI literacy events at schools. It covers the entire event: participant registration, a built-in learning Academy, secure file submissions, structured judging with weighted rubrics, and GDPR-compliant data handling.

We built it because running a good AI competition is hard work, and no decent tool existed for schools.

Why privacy is the foundation

Promptathon is designed to be entirely self-hosted, so data never leaves the institution responsible for it. During alpha we host it carefully on a VPS in the EU/UK in per-school logically isolated databases.

After alpha, any school can run Promptathon independently on their own server — with zero access from us.

FAQ

8 Entries
Q-01 Is it really free? +
Yes. No hidden costs, no premium tier, no expiry on alpha access. We're students building this to support AI education — making money is not the current goal.
Q-02 Do I need to run a server? +
Not during alpha. We host every instance — you contact us, we set it up, you get a link. No servers, no SSH, no configuration. Post-alpha, self-hosting is an option for full data sovereignty, via a single setup script on a cheap VPS.
Q-03 Can I use my own domain? +
Yes. By default we give you a subdomain like yourschool.promptathon.uk. Prefer your own domain? Mention it and we'll set it up — you point a DNS CNAME or A record at our server, which takes two minutes.
Q-04 Where is student data stored? +
During alpha: on a VPS in the EU/UK in a per-school logically isolated database. We treat all data as strictly confidential and never access it except when explicitly asked for support. Post-alpha self-hosted: entirely on your own server, with zero access from us. That's the long-term design goal.
Q-05 Is it GDPR compliant? +
Yes. Built-in data-request handling lets participants submit access, export, and deletion requests through a self-service portal. All requests are tracked and logged. No telemetry, no analytics, no external data sharing.
Q-06 How many students can it handle? +
Comfortably hundreds on a modest server. The platform uses SQLite with WAL mode for concurrent access and threaded Gunicorn workers. For 300+ simultaneous submissions we recommend at least a 2-core, 4 GB instance.
Q-07 What happens after alpha? +
We haven't fixed a commercial model. Alpha schools get plenty of notice before anything changes, and will always be credited for helping build this. The core software will remain source-available.
Q-08 Who built Promptathon? +
Students who believe AI literacy education should be accessible to every school, not only well-funded ones. We built the thing we wished existed and are sharing it freely while we work out what it wants to become.
