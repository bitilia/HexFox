02

About

Mission / Architecture

Every school should be able to run a great competition — regardless of budget.

What it is

Competition management software built for educational events at schools and institutions. It covers the entire event: participant registration, a built-in Learning Hub, secure file submissions, structured judging with weighted rubrics, and GDPR-compliant data handling.

We built it because running a good competition is hard work, and no decent free tool existed for schools.

Why privacy is the foundation

HexFox is designed to be entirely self-hosted, so data never leaves the institution responsible for it. During alpha we host it carefully on a VPS in the EU/UK, in per-school logically isolated databases.

After alpha, any school can run HexFox independently on their own server — with zero access from us.

FAQ

8 Entries
Q-01 Is it really free? +
Yes. No hidden costs, no premium tier, no expiry on alpha access. We're students building this to support education — making money is not the current goal.
Q-02 Do I need to run a server? +
Not during alpha. We host every instance — you contact us, we set it up, you get a link. No servers, no SSH, no configuration required. Post-alpha, self-hosting is an option for full data sovereignty via a single setup script on a cheap VPS.
Q-03 Can I use my own domain? +
Yes. By default we give you a subdomain like yourschool.hexfox.app. Prefer your own domain? Mention it and we'll set it up — you point a DNS CNAME or A record at our server.
Q-04 Where is participant data stored? +
During beta: on a VPS in the EU/UK in a per-school logically isolated database. We treat all data as strictly confidential and never access it except when explicitly asked for support. Post-alpha self-hosted: entirely on your own server, with zero access from us.
Q-05 Is it GDPR compliant? +
Yes. Built-in data-request handling lets participants submit access, export, and deletion requests through a self-service portal. All requests are tracked and logged. No telemetry, no analytics, no external data sharing.
Q-06 How many participants can it handle? +
Comfortably hundreds on a modest server. The platform uses SQLite with WAL mode for concurrent access and threaded workers. For 300+ simultaneous submissions we recommend at least a 2-core, 4 GB instance.
Q-07 What happens after beta? +
We haven't fixed a commercial model. Alpha schools get plenty of notice before anything changes, and will always be credited for helping build this. The core software will remain source-available.
Q-08 Who built HexFox? +
Students who believe good competitions should be accessible to every school, not only well-funded ones. We built the thing we wished existed and are sharing it freely while we work out what it wants to become.
